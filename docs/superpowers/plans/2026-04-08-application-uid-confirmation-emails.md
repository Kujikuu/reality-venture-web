# Application UID and Confirmation Emails Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a unique reference ID (UID) to every application and send confirmation emails to applicants after general and startup submissions.

**Architecture:** A migration adds a `uid` column to applications. The Application model auto-generates UIDs on creation using a `booted()` method (matching the Booking model pattern). Two new queued Mailables with Blade markdown templates are dispatched from the controller after existing admin email and Google Sheets sync.

**Tech Stack:** Laravel 11, PHPUnit, Blade Markdown Mail, Queue (database driver)

- [ ] **Step 3: Implement UID generation in Application model**

In `app/Models/Application.php`, add `'uid'` to the `$fillable` array, and add the `booted()` method and `generateUid()` static method after the `casts()` method:

Add to `$fillable` array (after `'type'`):
```php
'uid',
```

Add after the `casts()` method:
```php
protected static function booted(): void
{
    static::creating(function (Application $application) {
        if (empty($application->uid)) {
            $application->uid = static::generateUid();
        }
    });
}

public static function generateUid(): string
{
    $characters = 'ABCDEFGHJKLMNPQRSTUVWXYZ23456789';

    do {
        $uid = 'RV-';
        for ($i = 0; $i < 6; $i++) {
            $uid .= $characters[random_int(0, strlen($characters) - 1)];
        }
    } while (static::where('uid', $uid)->exists());

    return $uid;
}
```

Note: The character set excludes `0`, `O`, `1`, `I`, `L` to avoid ambiguity when reading/sharing.

namespace App\Mail;

use App\Models\Application;
use Illuminate\Bus\Queueable;
use Illuminate\Contracts\Queue\ShouldQueue;
use Illuminate\Mail\Mailable;
use Illuminate\Mail\Mailables\Content;
use Illuminate\Mail\Mailables\Envelope;
use Illuminate\Queue\SerializesModels;

class GeneralApplicationConfirmation extends Mailable implements ShouldQueue
{
    use Queueable, SerializesModels;

    public function __construct(public Application $application) {}

    public function envelope(): Envelope
    {
        return new Envelope(
            subject: 'Your Application to Reality Venture - ' . $this->application->uid,
        );
    }

    public function content(): Content
    {
        return new Content(
            markdown: 'mail.general-application-confirmation',
        );
    }

    /** @return array<int, \Illuminate\Mail\Mailables\Attachment> */
    public function attachments(): array
    {
        return [];
    }
}
